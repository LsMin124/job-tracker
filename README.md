# 채용 관제 보드 (Job Tracker)

로봇 · 생산 SW 신입 포지션 취업 준비용 채용공고 트래커.

**Live**: https://lsmin124.github.io/job-tracker/

## 기능

- 공고 카드 목록 — 기업 / 조직 / 직무 / 직무군 / 마감 D-day
- **3개 소스 공고 자동 수집** — GitHub Actions가 매일 06:00 KST에 ①자소설닷컴(등록 기업 24페이지 + 전체 검색 전수 열거) ②사람인(IT 직군 최신) ③링커리어(최신 채용)를 크롤링해 `data/postings.json` 갱신. 카드에 접수중 공고를 소스 라벨·마감 D-day와 함께 표시. **학사 신입 지원 가능 공고만 수집** — 경력·인턴·교육(채용형태), 석·박사 전용(제목 판별) 제외
- **중견 이상 전수 수집 → 자동 카드 등록** — 자소설 전체 검색에서 대기업·중견 신입 공고를 전부 수집하고(직무 키워드 필터 없음 — 전 직무·전 산업), 접수중 공고가 있는 기업을 모두 카드로 자동 등록. 규모 미상 소스(사람인·링커리어)는 공채형 제목만 채택해 중소 노이즈 차단
- **개인별 숨김** — 카드 「숨김」 버튼으로 목록에서 제외(툴바 「숨김 N」으로 열람·복원). 데이터가 브라우저별 저장이라 동료 각자 자신의 숨김·상태·우선순위를 따로 관리
- **공고 내 직무 리스트** — 한 공고의 신입 직무 전체(최대 40개)를 카드 안에 나열, 관심 키워드(로봇·제어·비전·SW·백엔드 등) 일치 직무는 강조 후 상단 정렬
- **우선순위 지정** — 카드별 P1(최우선)~P5(보류)를 수정 모달에서 지정 (P1·P2는 배지 강조)
- **정렬** — ① 현재 접수중 여부 ② 기업규모(대기업→중견→중소→기타) ③ 사용자 우선순위 ④ 마감 임박 순
- **JD·매칭·근거 상세** — 카드마다 JD 핵심 요약, 「요건 ← 내 근거」 매칭 표, 채용 페이지 링크를 접이식으로 표시
- **이력 요약 패널** — 프로젝트·논문·교육·자격을 상단에 고정, 카드 매칭의 참조 기준
- 전형 단계 진행바 (지원전 → 서류 → 시험 → 면접 → 결과)
- 상태 관리: 예정 · 서류작성 · 지원완료 · 코테·인적성 · 면접 · 최종합격 · 탈락 · 관망
- 직무군 / 진행 상태 필터
- 공고 추가 · 수정 · 삭제 — JD 요약, 매칭(`요건 :: 근거`), 링크(`라벨 :: URL`)도 줄 단위로 편집
- 마감 임박(7일 내) 카운터
- `localStorage` 자동 저장 — 서버 없음, 정적 페이지 단독 동작

카드의 고정 링크는 상시 유효한 채용 허브(공식 채용 사이트, 자소설 기업 페이지)만 사용합니다.
마감된 개별 공고 링크는 두지 않으며, 접수중 공고는 자동 수집 데이터가 실시간으로 대체합니다.
시드의 JD 요약은 공고 페이지 모집 직무 목록 실조회(2026-07 검증) 기반이며 사이트에서 수정 가능합니다.

## 자동 수집 (크롤러)

- `crawler/crawl.py` — 표준 라이브러리만 사용, 소스 3종:
  - **자소설닷컴**: 등록 기업 페이지 + 전체 검색(`division=1`, 마감 제외) 전수 열거, 기업규모로 대기업·중견 필터
  - **사람인**: IT 직군 최신 목록 HTML 파싱, career 텍스트로 신입 판별
  - **링커리어**: 목록 페이지 임베디드 데이터(Apollo state), jobTypes=NEW 판별
- 동일 기업·동일 마감일 공고는 자소설 우선으로 중복 제거. 개별 소스 장애는 전체 수집을 막지 않음
- 수집 출처는 기업 공식 홈페이지가 아닌 집계 사이트 — 공식 사이트는 클라이언트
  렌더링·구조 상이로 정적 크롤링이 어려워 단일 파서로 전 기업을 커버 가능한 경로를 택함
- `.github/workflows/crawl.yml` — 매일 06:00 KST 실행, 변경 시 자동 커밋 (수동 실행: Actions 탭 → crawl-postings → Run workflow)
- 두산로보틱스·한화로보틱스는 자소설 미등재로 자동 수집 미지원 — 카드의 공식 링크로 수동 확인
- 시드 기업 목록: 기존 13개사 + 확장 12개사(레인보우로보틱스·로보스타·고영테크놀러지·한화로보틱스·에스에프에이·기아·현대로템·HL만도·한화시스템·LG이노텍·SK하이닉스·삼성디스플레이) = 26장. 시드 변경 시 `SEED_VER`가 올라가며 기존 사용자에게 신규 카드가 자동 병합됨

## 사용

브라우저에서 `index.html`을 열거나 GitHub Pages 배포 페이지를 이용하면 됩니다.
빌드 도구, 의존성 없음 (Google Fonts CDN 제외).

데이터는 브라우저 `localStorage`(`jobtracker:v2` 키)에 저장됩니다. 구버전 데이터는
접속 시 자동 마이그레이션되어 상태·메모 수정이 보존됩니다.
「초기 목록으로 되돌리기」 버튼으로 시드 목록(26건)을 복원할 수 있습니다.
